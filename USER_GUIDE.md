# Campus Docs Assistant 用户使用手册

## 欢迎使用

欢迎使用 Campus Docs Assistant - AI产品经理一站式工作平台！本平台由好铁汁精心打造，专为大二学生设计，帮助你系统学习AI产品知识、掌握产品经理核心技能。

## 快速开始

### 系统要求
- 现代浏览器（Chrome 90+、Firefox 88+、Safari 14+、Edge 90+）
- 网络连接（用于访问后端服务）

### 访问平台
1. 确保后端服务已启动（默认地址：http://localhost:3001）
2. 打开浏览器访问前端地址（默认地址：http://localhost:5173）
3. 你将看到首页欢迎界面

## 核心功能模块

### 1. AI产品案例分析

**功能介绍**
好铁汁精心整理的AI产品案例库，帮你快速理解行业最佳实践。

**使用步骤**
1. 进入"AI产品案例分析"模块
2. 使用搜索框输入关键词（如：ChatGPT、Midjourney）
3. 使用筛选器选择类别（消费级/企业服务/垂直行业）或领域
4. 点击案例卡片查看详情
5. 查看完整分析：案例背景、产品功能拆解、AI技术应用、商业模式、优缺点评估、改进建议
6. 点击"收藏"按钮保存感兴趣的案例

**学习建议**
- 每周至少阅读2-3个案例
- 对比同类产品的不同实现方案
- 思考案例中的改进建议是否合理
- 尝试用自己的话总结案例要点

### 2. PRD模板库

**功能介绍**
好铁汁为你准备的标准化PRD模板，快速产出专业产品需求文档。

**使用步骤**
1. 进入"PRD模板库"模块
2. 选择模板类型：AI功能类、工具类、服务类
3. 点击"查看详情"查看模板内容
4. 切换标签页查看：模板内容、使用指南、撰写示例
5. 点击"下载模板"保存为Markdown文件
6. 在本地编辑器中修改使用

**模板说明**
- **AI功能类**：适用于智能推荐、对话系统、图像识别等AI功能产品
- **工具类**：适用于效率工具、数据工具、开发工具等实用型产品
- **服务类**：适用于在线教育、医疗健康、金融服务等平台型产品

**使用建议**
- 先阅读使用指南了解模板结构
- 参考撰写示例理解填写规范
- 根据实际项目需求调整模板内容
- 保持文档的完整性和一致性

### 3. 用户画像构建工具

**功能介绍**
好铁汁帮你精准定义目标用户，深入理解用户需求与行为特征。

**使用步骤**
1. 进入"用户画像构建"模块
2. 点击"新建画像"按钮
3. 填写基本信息：用户名称、年龄、性别、职业、学历
4. 描述行为特征：使用频率、偏好渠道、决策因素
5. 分析需求痛点：用户面临的主要问题和挑战
6. 设定用户目标：短期目标和长期目标
7. 选择使用场景：学习提升、工作办公、娱乐休闲等
8. 添加标签：使用推荐标签或自定义标签
9. 点击"保存"完成画像创建
10. 在列表中点击"查看"查看画像详情
11. 点击"导出报告"生成Markdown格式报告

**画像要素说明**
- **基本属性**：用户的人口统计学特征
- **行为特征**：用户的使用习惯和偏好
- **需求痛点**：用户面临的问题和挑战
- **用户目标**：用户期望达成的目标
- **使用场景**：用户使用产品的具体情境
- **标签**：快速识别用户特征的关键词

**使用建议**
- 为每个核心用户群体创建独立画像
- 基于真实用户调研数据填写信息
- 定期更新画像以反映用户变化
- 将画像分享给团队成员对齐认知

### 4. 产品原型设计基础

**功能介绍**
好铁汁带你掌握原型设计方法论，快速构建产品交互原型。

**使用步骤**

#### 学习设计方法论
1. 进入"产品原型设计"模块
2. 查看"设计方法论"标签页
3. 学习四大核心方法论：
   - 用户流程图：描述用户完成目标的步骤路径
   - 线框图设计：快速表达页面布局与信息架构
   - 交互设计原则：提升用户体验的核心设计准则
   - 移动端适配：多设备场景下的设计策略
4. 展开"核心要点"查看详细说明

#### 了解组件库规范
1. 切换到"组件库规范"标签页
2. 查看六大基础组件：按钮、表单、列表、卡片、导航、表格
3. 了解每个组件的设计规范和使用场景
4. 参考组件预览理解设计要点

#### 使用原型草图工具
1. 切换到"原型草图工具"标签页
2. 选择绘图工具：选择、矩形、圆形、线条、文本
3. 选择颜色和线宽
4. 在画布上绘制原型
5. 点击"清空画布"重新开始
6. 点击"导出原型"保存为PNG图片

**学习建议**
- 先掌握方法论再动手实践
- 从简单页面开始练习
- 参考优秀产品设计案例
- 多与团队成员讨论设计方案

### 5. 数据统计与分析

**功能介绍**
好铁汁通过数据驱动产品优化，持续改进用户体验。

**使用步骤**
1. 进入"数据统计"模块
2. 查看总体统计：总访问量、独立用户数、今日访问
3. 查看模块访问排行：各功能模块的使用情况
4. 查看用户行为分析：高频操作统计
5. 阅读产品优化洞察：基于数据的产品改进建议

**数据说明**
- **总访问量**：所有用户的访问次数总和
- **独立用户数**：不同用户的数量（基于用户ID）
- **今日访问**：当天的访问次数
- **模块访问排行**：各功能模块的访问次数、独立用户数、最后访问时间
- **用户行为分析**：用户在各个模块的具体操作统计

**使用建议**
- 定期查看数据统计了解使用情况
- 关注高频功能持续优化体验
- 分析低频功能找出改进机会
- 基于数据做出产品决策

## 常见问题

### Q1: 为什么后端服务无法连接？
A: 请检查后端服务是否已启动，默认端口为3001。确保防火墙未阻止连接。

### Q2: 数据保存在哪里？
A: 所有数据保存在本地SQLite数据库中（backend/db/campus.db），支持数据备份和恢复。

### Q3: 如何备份数据？
A: 可以复制backend/db/campus.db文件进行备份，或使用系统提供的数据导出功能。

### Q4: 支持多用户吗？
A: 当前版本为单用户模式，后续版本将支持多用户认证和数据隔离。

### Q5: 如何更新系统？
A: 拉取最新代码后，重新安装依赖并启动服务即可。数据库结构保持向后兼容。

### Q6: 案例数据可以自定义吗？
A: 可以，通过backend/seed.js文件修改案例数据，重新运行初始化脚本即可。

### Q7: 原型草图工具支持撤销吗？
A: 当前版本暂不支持撤销功能，建议定期导出保存。后续版本将添加此功能。

### Q8: 如何反馈问题或建议？
A: 可以通过GitHub Issues提交问题反馈和功能建议。
