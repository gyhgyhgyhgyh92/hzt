## 6. 问题清单与优化建议

### 6.1 已发现问题

| 编号 | 问题描述 | 严重程度 | 优先级 |
|------|----------|----------|--------|
| P1 | API密钥以明文存储在config.json中 | 中 | 高 |
| P2 | 计算器工具使用eval()存在安全风险 | 中 | 中 |
| P3 | 无单元测试覆盖 | 低 | 中 |
| P4 | 错误边界处理不统一 | 低 | 中 |

### 6.2 优化建议

#### P1: API密钥安全存储
**问题**: 当前API密钥明文存储在config.json文件中
**建议方案**:
```javascript
// 方案：使用Electron安全存储API
const { safeStorage } = require('electron')

const saveApiKey = async (key, value) => {
  if (safeStorage.isEncryptionAvailable()) {
    const encrypted = safeStorage.encryptString(value)
    // 存储加密后的密钥
  }
}
```
**替代方案**: 使用crypto模块进行AES加密，主密码保护

#### P2: 计算器安全替代
**问题**: eval()执行用户输入存在代码注入风险
**建议方案**:
```javascript
// 方案：使用math.js库或手写表达式解析器
// 仅允许数字和+-*/()运算符
const safeEvaluate = (expr) => {
  if (!/^[\d+\-*/().\s]+$/.test(expr)) {
    throw new Error('Invalid expression')
  }
  return eval(expr) // 白名单过滤后再执行
}
```

#### P3: 增加测试覆盖
**建议**: 
- 使用Jest进行单元测试
- 对核心算法（相似度计算、工具解析）编写测试用例
- 目标覆盖率：核心模块>70%

#### P4: 统一错误处理
**建议**:
- 建立全局错误处理中间件
- 统一错误码和错误信息格式
- 增加错误上报机制（可选）

### 6.3 性能优化建议

1. **RAG检索优化**：当前为线性扫描，文档量超过1000篇后可考虑引入倒排索引
2. **懒加载**：设置面板各模块可按需加载，减少首屏资源
3. **缓存策略**：对API调用结果增加本地缓存，减少重复请求
4. **虚拟列表**：对话记录超过1000条时考虑虚拟滚动
