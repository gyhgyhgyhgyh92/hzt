# Campus Docs Assistant 重构说明文档

## 项目概述

Campus Docs Assistant 是一个面向大二学生的AI产品经理一站式工作平台，通过系统化的功能模块帮助用户学习AI产品相关知识、掌握产品经理核心技能。

## 重构目标

1. **目标受众精准定位**：面向大二学生，融合AI产品经理实习岗位专业能力培养要素
2. **中文界面与品牌建设**：全系统采用中文界面，融入"好铁汁"个人品牌元素
3. **核心功能模块开发**：实现4大核心功能模块，满足学习需求并展示职业技能
4. **架构优化**：模块化设计，低耦合高内聚，支持功能扩展
5. **数据驱动**：实现使用数据统计与分析功能，体现产品数据分析能力

## 技术栈

### 前端
- Vue3 + Vite：现代化前端开发框架
- Element Plus：UI组件库
- Vue Router：路由管理
- Fetch API：HTTP请求

### 后端
- Express：Node.js Web框架
- sql.js：纯JavaScript SQLite实现（替代better-sqlite3解决编译问题）
- CORS：跨域支持

## 核心功能模块

### 1. AI产品案例分析模块
**需求背景**：大二学生在学习AI产品相关知识时，缺乏系统化的案例库和学习路径。

**核心功能**：
- 收录10+真实AI产品案例，覆盖消费级应用、企业服务、垂直行业解决方案
- 标准化分析模板：案例背景、产品功能拆解、AI技术应用分析、商业模式解读、优缺点评估、改进建议
- 案例搜索、分类筛选、收藏与笔记功能
- 访问统计埋点

**文件位置**：
- 前端：`frontend/src/views/CasesView.vue`
- 后端：`backend/routes/cases.js`

### 2. PRD模板库模块
**需求背景**：产品经理在撰写需求文档时，常面临格式不统一、结构不完整的问题。

**核心功能**：
- 3套差异化PRD模板：AI功能类、工具类、服务类
- 完整文档结构：产品概述、用户需求、功能需求、非功能需求、原型说明、数据指标、风险评估
- 模板使用指南、撰写示例及常见问题解答
- 模板下载功能
- 访问统计埋点

**文件位置**：
- 前端：`frontend/src/views/PrdView.vue`
- 后端：`backend/routes/prd.js`

### 3. 用户画像构建工具模块
**需求背景**：产品设计过程中，精准定义目标用户是确保产品成功的关键。

**核心功能**：
- 标准化用户画像模板：基本属性、行为特征、需求痛点、用户目标、使用场景
- 智能标签推荐功能，支持自定义标签创建
- 用户画像可视化展示
- 用户画像报告导出（Markdown格式）
- 访问统计埋点

**文件位置**：
- 前端：`frontend/src/views/PersonaView.vue`
- 后端：`backend/routes/persona.js`

### 4. 产品原型设计基础模块
**需求背景**：原型设计是产品经理的核心技能之一，但初学者往往缺乏系统的方法论指导和实践工具。

**核心功能**：
- 原型设计方法论知识库：用户流程图、线框图、交互设计原则、移动端适配
- 基础组件库说明文档：按钮、表单、列表、卡片、导航、表格等常用UI组件
- 简易原型草图工具：支持矩形、圆形、线条、文本绘制，画布导出
- 访问统计埋点

**文件位置**：
- 前端：`frontend/src/views/PrototypeView.vue`
- 后端：`backend/routes/prototype.js`

### 5. 数据统计与分析模块
**需求背景**：通过数据驱动产品优化，持续改进用户体验。

**核心功能**：
- 总体统计：总访问量、独立用户数、今日访问
- 模块访问排行：各功能模块访问次数、独立用户、最后访问时间
- 用户行为分析：高频操作统计
- 产品优化洞察：基于数据的产品改进建议

**文件位置**：
- 前端：`frontend/src/views/StatsView.vue`
- 后端：`backend/routes/stats.js`
- 工具：`frontend/src/utils/stats.js`

## 技术亮点

### 1. sql.js替代better-sqlite3
**问题**：better-sqlite3在Node v24环境下因C++20编译失败。

**解决方案**：使用纯JavaScript实现的sql.js，确保跨平台兼容性和部署便捷性。

**影响**：虽然性能略有牺牲，但解决了编译问题，确保项目可正常运行。

### 2. 统一统计埋点系统
**设计思路**：
- 创建统一的统计工具函数 `frontend/src/utils/stats.js`
- 提供 `trackPageView`、`trackButtonClick`、`trackFeatureUse` 等便捷方法
- 后端提供 `/api/stats/track` 接口记录用户行为
- 支持模块访问统计、用户行为分析、总体统计查询

**实现效果**：
- 所有功能模块集成访问统计
- 数据驱动产品优化决策
- 体现产品数据分析能力

### 3. 需求背景说明
**设计思路**：
- 在每个核心功能模块添加"需求背景"说明
- 使用 Element Plus 的 Alert 组件展示
- 说明模块设计的用户需求和使用场景

**实现效果**：
- 体现产品思维：从用户需求出发设计功能
- 帮助用户理解模块价值
- 展示产品经理的需求分析能力

## 部署说明

### 环境要求
- Node.js >= 16.x
- npm >= 8.x

### 安装步骤

1. **安装后端依赖**
```bash
cd backend
npm install
```

2. **安装前端依赖**
```bash
cd frontend
npm install
```

3. **初始化数据库**
```bash
cd backend
node seed.js
```

4. **启动后端服务**
```bash
cd backend
node server.js
```
后端服务将在 `http://localhost:3001` 启动

5. **启动前端开发服务器**
```bash
cd frontend
npm run dev
```
前端服务将在 `http://localhost:5173` 启动

### 生产环境部署

1. **构建前端**
```bash
cd frontend
npm run build
```

2. **启动生产服务**
```bash
cd backend
NODE_ENV=production node server.js
```

访问 `http://localhost:3001` 即可使用完整应用。
